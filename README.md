# UPI Payment Failure Analysis System

## 📌 Project Overview
This project focuses on analyzing UPI transaction failures to identify patterns, improve success rates, and streamline the refund process. As a Business Analyst, I've covered the end-to-end journey from requirement gathering to data-driven insights.

## 🛠️ Project Deliverables
* **Business Requirements Document (BRD):** Detailed documentation of stakeholder needs and system scope.
* **Process Mapping:** As-Is and To-Be flows created to visualize the optimized refund journey.
* **SQL Data Analysis:** Advanced queries to extract failure rates, bank-wise performance, and SLA breaches.
* **Interactive Power BI Dashboard:** A visual report tracking a 56% success rate, top failure reasons (Bank Timeouts), and refund statuses.

## 📊 Key Insights
* **Primary Failure Reason:** "Bank Timeout" accounts for the majority of failures, indicating a need for better bank server integration.
* **Bank Performance:** HDFC showed the highest volume of failures in Q1 2026.
* **Operational Goal:** Identified a path to reduce refund processing time from 3 days to same-day.
